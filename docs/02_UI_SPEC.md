# 02. UI仕様書: レトロゲーム風

## 1. デザイン方向

採用方向は**レトロゲーム風**です。参考画像は`assets/retro_ui_concept.png`です。

参考画像は雰囲気と画面構成を示すコンセプトです。完全な模写はせず、スマートフォンの可読性、軽量性、広告ポリシーへの配慮を優先します。

## 2. デザイン原則

1. 昔のゲームセンターを連想させる温かいレトロ感を出す。
2. 特定のゲーム筐体、競馬団体、公式サービスを模倣しない。
3. ギャンブル臭を抑える。オッズ、払戻、馬券、ジャックポット、大当たり、激アツ、点滅、過度なネオンを使わない。
4. 馬は番号のみで識別する。
5. 推し馬指定を使った場合は画面上で明示する。
6. ゲーム操作、カンパ、広告を視覚的に明確に分離する。

## 3. 暫定文言

| 項目 | 文言 |
|---|---|
| タイトル | `競馬ランダム抽選` |
| 副題 | `レトロ競走シミュレーター` |
| 主ボタン | `レースを開始する` |
| スキップ | `結果を見る` |

## 4. カラートークン

| 要素 | 推奨色 | 用途 |
|---|---|---|
| 背景 | 温かみのあるクリーム | 全体 |
| 濃色 | 深いフォレストグリーン | ヘッダー、枠 |
| 主ボタン | 落ち着いた深紅 | 開始、再実行 |
| 副ボタン | 落ち着いたグリーン | 設定変更 |
| アクセント | 控えめなゴールド | 星、トロフィー |
| 本文 | 濃いブラウンまたは黒 | 可読性優先 |

色だけで状態を伝えず、星、ラベル、文言を併用します。点滅演出は使いません。

## 5. 画面遷移

```text
設定画面
  -> レース画面
  -> 結果画面
       -> 同じ設定で新しいレース
       -> 頭数・推し馬を変更
       -> 外部カンパページを開く
```

## 6. 設定画面

```text
[ レトロ風タイトル ]
[ 出走頭数カード ]
[ 推し馬指定カード ]
[ レース開始ボタン ]
[ 前回結果カード ]
[ 注意文 ]
```

### 出走頭数カード

| 要素 | 動作 |
|---|---|
| 見出し | `出走頭数` |
| マイナス | 1頭減らす。2頭で無効 |
| 数値入力 | 初期値12。2〜18の整数だけ許可 |
| プラス | 1頭増やす。18頭で無効 |
| クイック選択 | 5頭、8頭、12頭、18頭 |

### 推し馬指定カード

| 要素 | 動作 |
|---|---|
| 見出し | `推し馬指定` |
| トグル | 初期値OFF |
| 補足 | `注目している番号を1着に固定できます` |
| 番号一覧 | ONの場合のみ1〜出走頭数を表示 |
| 選択 | 選択番号を強調し、小さな星を付ける |
| 注意 | `※ONの場合、選択した番号が1着になります` |

ONかつ未選択では、開始ボタンを無効にして選択を促します。

### 前回結果

`localStorage`の保存結果がある場合だけ、上位5頭、頭数、日時を表示します。保存に失敗した場合はカードを表示しません。

### 広告

初期画面の下に、ゲーム設定と開始ボタンから十分に距離を取って表示します。

## 7. レース画面

```text
[ レース開催中 ] [ 結果を見る ]
[ 番号付き横長レーン ]
[ ゴール ]
[ 実況カード ]
```

- 2〜18頭を横方向のレーンに並べる。
- 頭数に応じてレーン高と馬画像を縮小する。
- 推し馬は星と控えめなレーン強調で示す。
- 中間順位は演出として変えるが、最終着順は開始時に確定済みとする。
- `prefers-reduced-motion: reduce`では動きを短縮または抑制する。
- レース中は可能な限り縦スクロールを発生させず、18頭の番号を判読できるようにする。

実況例:

- `各馬一斉にスタートしました！`
- `7番が先頭に立ちました！`
- `最後の直線、接戦です！`

## 8. 結果画面

```text
[ レース結果 ]
[ 1〜5着カード ]
[ シミュレーション結果メモ: ◎○▲△× ]
[ 推し馬指定の説明: 使用時のみ ]
[ もう一度レースをする ]
[ 頭数・推し馬を変更する ]
[ カンパカード: URL設定時のみ ]
```

- 1着を大きく表示する。
- 2〜5着を一覧表示する。
- `◎○▲△×`は主表示ではなく、`シミュレーション結果メモ`として補助表示する。
- 推し馬指定時だけ`選択した番号を1着として表示しています`と明示する。
- リプレイは同じ設定を使うが、結果は新しく生成する。

## 9. カンパカード

URL設定時だけ表示します。

```text
[ このゲームを応援する ]
カンパの有無で抽選結果や機能は変わりません。
```

- 広告に見えるデザインにしない。
- カンパで結果が変わると誤解させない。
- カンパと広告の間に十分な余白と区切りを置く。

## 10. 広告エリア

- 初期画面の下に置く。
- 本番の見出しは`広告`とする。
- 開発時だけ`広告エリア`プレースホルダーを表示する。
- ゲーム画面、レース開始、リプレイ、設定変更、カンパから十分に離す。
- Auto adsが意図しない位置へ表示されないか公開前に確認する。

## 11. レスポンシブ・アクセシビリティ

| 条件 | ルール |
|---|---|
| 360〜430px | 主対象。横スクロールなし |
| 431〜767px | モバイル構成を維持 |
| 768px以上 | 最大幅560px程度で中央寄せ |

- タップ領域は原則44 CSS px以上。
- キーボードフォーカスを可視化する。
- アイコンだけのボタンに`aria-label`を付ける。
- 実況には`aria-live="polite"`を検討する。
- 馬画像が読み込めなくても番号と結果は読めるようにする。
