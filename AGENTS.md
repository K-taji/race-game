# AGENTS.md

## 1. 最初に読むこと

実装前に`README.md`と`docs/`配下の全ファイルを読んでください。`確定`、`暫定決定`、`必須`、`公開前入力`を区別してください。

## 2. 実装対象

- 新版は`public_html/race-game/`へ配置する独立した静的Webアプリです。
- HTML、CSS、JavaScript、画像、設定ファイルで実装してください。
- WordPressテーマ、プラグイン、`functions.php`は原則変更しないでください。
- 開発時はローカルHTTPサーバーで確認してください。`file://`での動作確認だけで済ませないでください。

## 3. 勝手に追加しないこと

- 会員登録、データベース、サーバーサイドAPI、実際の競馬データAPI、独自決済処理を追加しない。
- 実在馬名、公式ロゴ、実在馬写真、既存ゲームの模倣画像を使わない。
- 的中率向上、必勝、勝利確率向上、大当たりなど、誤認を招く表現を追加しない。
- カンパサービスを特定事業者へハードコードしない。
- AdSenseコードを抽選ロジックやUI状態管理へ直接埋め込まない。
- リプレイごとに広告を自動更新しない。
- AI開発用資料、原本、バックアップを`public_html`へ置かない。

## 4. MVP仕様

- スマートフォン優先。レトロゲーム風だが、カジノ、パチンコ、馬券売場のような射幸的表現は避ける。
- 出走頭数は2〜18頭。初期値は12頭。クイック選択は5頭、8頭、12頭、18頭。
- 数値入力、プラス、マイナス、クイック選択を用意する。
- 推し馬指定は任意で初期値OFF。ONの場合は1頭を選択し、指定番号を必ず1着にする。
- 通常モードでは全着順をランダム生成する。
- リプレイ時は同じ設定で新しい結果を生成する。前回結果を再利用しない。
- 結果は1〜5着と`◎○▲△×`を補助表示する。
- 馬は簡単なアニメーションで走らせ、中間順位の変動演出を入れる。
- レース時間は8〜12秒。`結果を見る`でスキップできる。
- 短い実況テキストを表示する。初期版では効果音を使わない。
- 前回結果は`localStorage`へ1件保存する。保存に失敗してもゲームを継続できる。

## 5. 抽選ロジック

- 最終着順はレース開始時に確定する。
- 最終着順の乱数には`crypto.getRandomValues()`を使う。
- `Math.random()`を最終着順の生成に使わない。
- シャッフルは偏りを避ける実装にする。
- DOM上の表示位置から最終結果を決めない。

## 6. 外部連携

- カンパは`config/app-config.js`から設定する。URL未設定時はカンパカードを非表示にする。
- カンパURLはHTTPSのみを許可する。外部リンクには`target="_blank"`と`rel="noopener noreferrer"`を付ける。
- AdSenseは`assets/js/integrations/adsense.js`へ分離する。
- AdSenseの手動広告ユニットは、結果画面下の専用枠へ初回のみマウントする。リプレイで再マウント・自動更新しない。
- 開発時は`広告エリア`プレースホルダーを表示する。
- GA4は`assets/js/integrations/analytics.js`へ分離する。静的アプリ側で`gtag`またはGTMの設定を確認する。

## 7. 広告配置

- 広告は操作ボタン、カンパボタン、ゲーム画面と明確に分離する。
- 広告見出しは`広告`とする。応援、スポンサー、おすすめリンクなど、誤解を招く見出しを使わない。
- Auto adsが既存サイトで有効な場合、`/race-game/`ではページ除外を検討し、手動広告ユニットを優先する。
- 公開前にスマートフォンで意図しない広告位置がないか確認する。
