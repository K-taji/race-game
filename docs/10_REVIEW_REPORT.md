# 10. 厳重レビュー報告

## 1. 結論

旧版0.3は、そのままAIへ渡してはいけません。直近要望と矛盾する旧構成が混在していました。本版0.5で修正済みです。

## 2. 発見した問題と対応

| ID | 重要度 | 旧版の問題 | 0.5での対応 |
|---|---|---|---|
| R-01 | 致命的 | WordPressページ、`functions.php`、ショートコード案が残っていた | `public_html/race-game/`の独立静的アプリへ統一 |
| R-02 | 致命的 | カンパ以外の項目がTBDのまま | 暫定決定へ変更し、MVPで実装対象にした |
| R-03 | 高 | カンパURLが単一定数で、拡張設計が弱い | 設定ファイルとproviderアダプターへ分離 |
| R-04 | 高 | AdSenseの後付け方法が不足 | config、専用連携層、HEADマーカー、手動枠を定義 |
| R-05 | 高 | リプレイ時の広告再読込ルールがなかった | 1ページビュー1回だけマウント。自動更新禁止 |
| R-06 | 高 | WordPress側GA4が静的HTMLへ継承される前提が残っていた | 静的アプリ側でgtag/GTMを設定するルールを追加 |
| R-07 | 高 | AI資料や原本を公開する事故への対策がなかった | 本番アップロード対象を`index.html`、`assets/`、`config/`だけに限定 |
| R-08 | 中 | `/race-game`と物理フォルダURLの扱いが曖昧 | 正規URLを`/race-game/`へ統一し、転送確認を追加 |
| R-09 | 中 | Auto adsが操作付近へ出るリスクが弱かった | ページ除外の確認と手動配置優先を追加 |
| R-10 | 中 | リプレイが前回結果を再利用する誤読余地があった | 同じ設定で新しい着順を生成すると明示 |
| R-11 | 中 | 18頭表示を幅だけで確認していた | 画面高を含む端末サイズでテストするよう変更 |
| R-12 | 中 | reduced motion、localStorage障害、画像失敗の扱いが不足 | フォールバックとテストを追加 |
| R-13 | 中 | 射幸的デザインを避ける要望が薄かった | 禁止表現、禁止視覚要素をUI仕様へ追加 |
| R-14 | 中 | ads.txt、プライバシーポリシー、CMPの確認がなかった | 公開前チェックへ追加 |
