# 01. 要件定義

## 1. 要件の扱い

原本でチェックが付いた項目と、チャットで追加・確定した項目を確定要件とします。原本で未選択だった項目は、カンパサービス以外をMVP向けに暫定決定します。

## 2. 機能要件

### FR-01 出走頭数

| 要件 | 内容 | ステータス |
|---|---|---|
| 最小頭数 | 2頭 | 確定 |
| 最大頭数 | 18頭 | 確定 |
| 入力方法 | 数値入力、プラス、マイナス | 暫定決定 |
| クイック選択 | 5頭、8頭、12頭、18頭 | 暫定決定 |
| 初期値 | 12頭 | 暫定決定 |
| 検証 | 2〜18の整数だけを受け付ける | 必須 |

### FR-02 レースモード

| モード | 動作 | ステータス |
|---|---|---|
| 通常モード | 全着順をランダムに生成する | 必須 |
| 推し馬指定モード | 任意で1頭指定し、指定番号を必ず1着にする。残りはランダム生成する | 確定 |

推し馬指定ルール:

1. 初期値はOFF。
2. OFFでは指定なしで開始できる。
3. ONでは1頭だけ選択できる。
4. ONかつ番号未選択では開始ボタンを無効化する。
5. 頭数変更で指定番号が範囲外になった場合は選択を解除する。
6. 設定画面に`※ONの場合、選択した番号が1着になります`と明示する。
7. 結果画面にも推し馬指定を使用したことを明示する。
8. 的中率向上、必勝などの誤解を招く文言を使わない。

### FR-03 抽選結果

| 要件 | 内容 | ステータス |
|---|---|---|
| 重複防止 | 同じ番号を重複表示しない | 必須 |
| 範囲 | 1〜出走頭数の範囲内 | 必須 |
| 確定時点 | レース開始時に最終着順を確定する | 必須 |
| 乱数 | `crypto.getRandomValues()`を使う。`Math.random()`を最終着順に使わない | 必須 |
| リプレイ | 同じ設定で新しい着順を生成する。前回着順を再利用しない | 必須 |
| 結果表示 | 1〜5着を表示する | 暫定決定 |
| 補助表示 | 上位5頭を`◎○▲△×`で併記する | 暫定決定 |

### FR-04 レース演出

| 項目 | 内容 | ステータス |
|---|---|---|
| 時間 | 8〜12秒 | 確定 |
| デザイン | レトロゲーム風 | 確定 |
| 馬 | 簡単なアニメーション | 暫定決定 |
| 順位変動 | 中間演出として入れる | 暫定決定 |
| 実況 | 短文表示 | 暫定決定 |
| スキップ | `結果を見る`で即時結果表示 | 暫定決定 |
| 効果音 | 初期版では使用しない | 暫定決定 |
| Reduced motion | 端末設定に応じ、動きを短縮または抑制する | 必須 |

### FR-05 前回結果

| 項目 | 内容 | ステータス |
|---|---|---|
| 保存 | `localStorage`へ1件 | 暫定決定 |
| 内容 | 頭数、モード、推し馬、上位5頭、日時 | 暫定決定 |
| 例外 | 保存失敗時もゲームを継続する | 必須 |

### FR-06 カンパ

| 要件 | 内容 | ステータス |
|---|---|---|
| 導線 | 任意の応援導線を設ける | 確定 |
| 表示位置 | 結果画面の操作ボタン下 | 暫定決定 |
| サービス | OFUSE、Stripe Payment Links、その他から後で選定 | 公開前入力 |
| URL | 外部サービスURL | 公開前入力 |
| 決済 | 自サイトでカード情報、支援額、決済完了状態を扱わない | 必須 |
| 結果への影響 | 一切影響させない | 必須 |
| 支援者特典 | 初期版では設けない | 暫定決定 |
| 拡張性 | URL、文言、表示ON・OFF、providerを設定ファイルで変更できる | 必須 |
| URL安全性 | HTTPS URLだけを許可する | 必須 |

### FR-07 AdSense

| 要件 | 内容 | ステータス |
|---|---|---|
| 既存AdSense | 継続する | 確定 |
| 表示位置 | 結果画面の下 | 確定 |
| 開発時 | `広告エリア`プレースホルダー | 必須 |
| 本番時 | 専用連携層から手動広告ユニットを表示 | 必須 |
| 表示回数 | 1ページビューにつき1回だけ初期化。リプレイで広告を自動更新しない | 必須 |
| 分離 | ゲーム、操作ボタン、カンパから明確に分離する | 必須 |
| Auto ads | 有効な場合は`/race-game/`の除外を検討し、手動配置を優先する | 必須確認 |

### FR-08 GA4

| 要件 | 内容 | ステータス |
|---|---|---|
| 既存状態 | WordPressサイトでは導入済み | 確定 |
| 静的アプリ | WordPress側のGA4が自動継承されると仮定しない | 必須 |
| 連携 | `analytics.js`へ分離し、未設定時もゲームを継続する | 必須 |

| イベント | 発火タイミング |
|---|---|
| `game_start` | レース開始 |
| `game_complete` | 結果画面表示。1レースにつき1回 |
| `replay` | もう一度レース |
| `skip_animation` | 演出スキップ |
| `runner_count_select` | 頭数変更 |
| `donation_button_click` | カンパボタン押下 |
| `favorite_horse_toggle` | 推し馬指定ON・OFF |
| `favorite_horse_select` | 推し馬番号選択 |

### FR-09 公開文言

画面内に簡潔に表示する:

- 本アプリは番号を使ったエンターテインメント型シミュレーターである。
- 推し馬指定モードでは、指定番号が1着になる。
- 表示結果は実際のレース結果を保証しない。
- カンパの有無は結果に影響しない。

## 3. 非機能要件

| ID | 要件 |
|---|---|
| NFR-01 | スマートフォン優先。360px幅で操作できる |
| NFR-02 | ページ全体の横スクロールを発生させない |
| NFR-03 | 設定・結果画面は縦スクロール可。レース中は18頭でも視認性を優先し、可能な限り1画面に収める |
| NFR-04 | タップ領域は原則44 CSS px以上 |
| NFR-05 | 文字と主要UIは十分なコントラストを確保する |
| NFR-06 | `prefers-reduced-motion`へ配慮する |
| NFR-07 | 画像を最適化し、軽量にする |
| NFR-08 | MVPでは個人情報を収集しない |
| NFR-09 | WordPressテーマ、プラグイン、`functions.php`へ依存させない |
| NFR-10 | AI開発用資料、原本、バックアップを公開領域へ置かない |
| NFR-11 | カンパ、AdSense、GA4を外部連携層へ分離する |
| NFR-12 | `race-game/`フォルダ単位で切り戻せる |
